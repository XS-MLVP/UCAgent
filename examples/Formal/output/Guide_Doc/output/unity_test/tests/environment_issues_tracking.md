# Traffic Light Controller 验证环境问题跟踪文档

## [ENV-001] 属性名：CK_TIMER_SHORT_VALID
- **状态**：待分析
- **失败类型**：TRIVIALLY_TRUE
- **问题分类**：待定
- **分析说明**：该属性断言 `timer_state` 与 `sh_t` 的逻辑关系，而这两者在 RTL 中是直接映射关系。工具通过常量传播直接证明了其为真。
- **修复措施**：待确定。通常 TRIVIALLY_TRUE 如果不是因为过度约束（Assume），可以保留，但需确认 `timer_state` 是否可达。由于 Cover 属性已通过，说明状态可达，此属性为真逻辑证明。

## [RTL-001] 属性名：CK_HWY_R_TO_G
- **状态**：已分类
- **失败类型**：FAIL
- **问题分类**：RTL缺陷
- **分析说明**：在 hwy_control 模块中，当状态为 RED 且收到 enable_hwy 信号时，状态转移赋值错误地写成了 `hwy_light <= RED`，正确应为 `GREEN`。这导致高速公路灯永远无法从红灯变回绿灯。
- **修复措施**：需修复 RTL 代码。

## [RTL-002] 属性名：CK_LIVE_FARM_GREEN
- **状态**：已分类
- **失败类型**：FAIL
- **问题分类**：RTL缺陷
- **分析说明**：由于 [RTL-001] 的存在，系统可能进入死锁或循环状态，导致农场灯无法正常轮转回绿灯（如果系统卡在高速公路红灯状态）。
- **修复措施**：随 [RTL-001] 一并修复。

## [RTL-003] 属性名：CK_LIVE_HWY_GREEN
- **状态**：已分类
- **失败类型**：FAIL
- **问题分类**：RTL缺陷
- **分析说明**：同上，高速公路灯由于代码逻辑错误无法回到 GREEN。
- **修复措施**：随 [RTL-001] 一并修复。
