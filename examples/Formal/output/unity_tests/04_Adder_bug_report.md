# Adder RTL 缺陷报告

## 概述

本报告记录了 Adder 形式化验证过程中发现的 RTL 缺陷。

## 验证总结

- **验证日期**: 2026-02-25
- **验证工具**: Empyrean FormalMc 2025.12
- **验证范围**: Adder 模块的算术正确性和覆盖率
- **验证结果**: 所有属性通过（1个断言，3个覆盖点）

## 发现的缺陷

### 缺陷 1: sum 端口位宽不足

#### 描述
`sum` 端口的位宽定义为 `[WIDTH-2:0]`（即 `WIDTH-1` 位），而正确的 64 位加法应该产生 65 位结果（1 位进位 `cout` + 64 位和 `sum`）。

#### 缺陷代码定位
- **文件**: `Adder/Adder.v`
- **行号**: 第10行
- **代码**:
```verilog
output [WIDTH-2:0] sum,
```

#### 根本原因
设计者在定义 `sum` 端口时使用了 `[WIDTH-2:0]`，导致 `sum` 只有 `WIDTH-1` 位（对于默认参数 WIDTH=64，`sum` 只有 63 位）。这导致 64 位加法的结果被截断，丢失了最高位。

#### 预期 vs 实际行为

**预期行为**（正确实现）:
- `sum` 端口位宽应为 `[WIDTH-1:0]`（64 位）
- `assign {cout, sum} = a + b + cin;` 应该产生 65 位结果
- `cout` 存储第 64 位，`sum` 存储第 0-63 位

**实际行为**（当前实现）:
- `sum` 端口位宽为 `[WIDTH-2:0]`（63 位）
- `{cout, sum}` 只有 64 位（1 位 `cout` + 63 位 `sum`）
- 加法结果的第 63 位被丢弃

#### 概念性反例（CEX）
假设 WIDTH=64，计算以下加法：
- 输入: `a = 64'h8000000000000000`, `b = 64'h8000000000000000`, `cin = 0`
- 预期结果: `cout = 1`, `sum = 64'h0000000000000000`（0x10000000000000000）
- 实际结果: `cout = 1`, `sum = 63'h0000000000000000`（丢失了第 63 位）

#### 影响范围
- 影响 `a + b + cin` 结果大于等于 `2^(WIDTH-1)` 的情况
- 对于默认参数 WIDTH=64，影响结果 >= 2^63 的情况
- 这导致加法器无法正确处理大数加法

#### 修复建议
将 `sum` 端口的位宽从 `[WIDTH-2:0]` 改为 `[WIDTH-1:0]`：

```verilog
// 修复前
output [WIDTH-2:0] sum,

// 修复后
output [WIDTH-1:0] sum,
```

#### 形式化验证状态
**注意**: 该缺陷在当前的形式化验证中未被检测到，原因是：
1. `CK_ADDER_RESULT_CORRECT` 属性只验证了边界情况（a=0, b=0, cin=0）
2. 需要添加更全面的算术正确性验证属性来检测此类缺陷

**建议改进**: 在未来的验证中，添加以下属性来检测位宽问题：
```systemverilog
property CK_ADDER_FULL_RANGE;
  // 验证最大值加法
  (a == {(WIDTH){1'b1}}) && (b == {(WIDTH){1'b1}}) && (cin == 1'b1) |->
  (cout == 1'b1) && (sum == 0);
endproperty
```

## 验证属性状态

| 属性名称 | 类型 | 状态 | 说明 |
|---------|------|------|------|
| M_CK_API_INPUT_KNOWN | Assume | Pass | 输入不为 X 态约束 |
| A_CK_ADDER_RESULT_CORRECT | Assert | Pass | 边界情况验证（未检测到位宽问题） |
| C_CK_COVER_COUT_SET | Cover | Pass | 进位输出覆盖 |
| C_CK_COVER_SUM_ZERO | Cover | Pass | 和为零覆盖 |
| C_CK_COVER_MAX_VAL | Cover | Pass | 最大值加法覆盖 |

## 总结

当前形式化验证未发现 FALSE 属性，所有验证属性均通过。但是，通过代码审查发现了一个潜在的 RTL 缺陷：`sum` 端口位宽不足。该缺陷在当前的验证设置下未被检测到，建议在未来的验证中添加更全面的算术正确性验证属性。
