# Bug 报告文档生成

## 概述

自动化生成 `{OUT}/04_{DUT}_bug_report.md`，从环境分析文档提取 RTL_BUG 属性，预填 BG 条目空壳，LLM 只需在 `[LLM-TODO]` 处填写根因分析、修复建议等。

## 步骤

### 1. 生成报告框架

调用工具 `InitBugReport`：

```python
InitBugReport(dut_name="{DUT}")
```

工具会自动从 `07_{DUT}_env_analysis.md` 提取 RTL_BUG，生成报告框架。无 RTL_BUG 时生成无缺陷声明。

### 2. 填写分析内容

打开生成的文档，逐条填写每个 `[LLM-TODO]` 标记处的内容：
- 所有 `<FG-*>` 和 `<FC-*>` 标签必须来自 `03_{DUT}_functions_and_checks.md`，用于对当前缺陷进行分类归属。
- **问题描述 / 根本原因 /预期行为 / 实际行为**：尽可能详细地解释产生 Bug 的代码机理。
- **反例波形解读**：请不要只用一两句话。应当按照给定的列表结构，分点写出触发条件、关键输入信号的值、预期输出以及实际输出。
- **根因分析总结**：如果在报告中有多个 Bug（如几十个属性失败），你需要在这里提炼它们是否源于同一个底层 RTL 代码错误。填写总结表格，并在一句话内给出统一的修复方案（例如“只需修改 Adder.v 第10行的位宽即可修复全部缺陷”）。

### 3. 完成后调用 Complete
